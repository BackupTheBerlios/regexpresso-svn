Introduction

This tool is made for programmers. It can help to test a regular expression prior to use it in some code.
It has not as many features as in professional tools, since it is only designed for "quick and dirty" tests.

Depending on your computer's capacities, the size of the subject to parse and the complexity of the regular expression, it may take more or less time to process the expression.
Be sure to use decently sized expressions and decent amounts of text.

Quick start

  1. Open RegExpresso or add it to your browser's sidebar by choosing one of the logos on top of this page depending on your browser
  2. Enter the text to parse in the top text area
  3. Click on the search button
  4. Enter a regular expression in the search field
  5. Validate by clicking the EVALUATE button

Example

Copy and paste this page's content into the text field, type the following pattern : \bE\w*\b in the search field to match all word beginning with a 'E'.

Browser requirements

  • The regular expressions flavour is the one of Javascript 1.2, which is similar to Perl's one.
  • Javascript 1.5 is needed (because of exception handling).
  • DOM Level 2 support
  • CSS 2 compatible
  • A fast computer, since there are some intensive calculations.
    For instance, I have found Konqueror requiring several explicit confirmations to continue executing the script because it was too slow (heavy load) on a Pentium III 450 MHz with 128 MB SDRAM. Of course, there was a large amount of data.

Known bugs and limitations

  • The Javascript RegExp class has some limitations as stated in the following article : http://www.webreference.com/js/column5/modifiers.html
  • Dubious reliability with browsers other than IE and Firefox ; lacks testing on that area.
  • The behaviour of the menu's buttons greatly depends on the browser and the windows context : whether RegExpresso is located in the sidebar, in the main window or in a separate window.
  • More bugs at berlios.de

Regular expressions syntax

The regular expression you are supposed to type in the pattern field follows the following syntax : command/options/modifiers.
Two commands are supported : "match" (m// operator, or an empty string : //) and "replace" (s/// operator).

  • To match a regular expression on some text, use the following syntax : m/pattern/modifiers or /pattern/modifiers,
  • To perform a substitution, type : s/pattern/replace/modifiers.
where :
  • pattern is the regular expression pattern
  • modifiers is a sequence of characters within {'g','i','m'}
  • replace is a substitution pattern
Don't forget the syntax depends on the accuracy of the browser's implementation of the Javascript's RegExp object.
I've found this page and the following to be good advice for Javascript-specific rules about regular expressions.

Options

The options panel can be switch on and off by clicking the OPTIONS button. One can click on the question mark next to each option to have a brief description.

Evaluate on the fly

When checked, the result is automatically updated whenever an option or the regular expression has changed. No need to click the EVALUATE button.

Display mode

The result can be displayed in 2 ways : text and table. Choose the one that fits your need.

In text mode, the full text is output, and the parts matched or replaced are highlighted. Other informations like indexes and groups are inserted before and after the highlighted parts. This mode is usefull when you want to have a visual overview of a selection or a replacement. Since the original formatting is kept, the text may overflow the window, and you'll then have to scroll.

In table, all informations are put in a table. This mode is usefull if you want to export the results in a program-ready layout. This mode is also usefull to display RegExpresso in a sidebar, since it has a column layout. This mode is not available for the replace function.

Show empty matches

Sometimes, a regular expression will lead to have empty results. In this case, checking this option makes them visible by replacing them with the "empty set" character : ∅ (may not be correctly rendered here). Otherwise, they are simply ignored (not displayed at all).

Show indexes

Each match has a number (starting from 0), called index.

Show groups

Groups (also named "back references") are parts of matches captured by parentheses in the regular expression pattern. They are very important to make something usefull out of a regular expression. There may be zero, one or more groups per match.

Show context

Check this option to display the surrounding context of every part that matched. Usefull to quickly find where each part was located in the original text. The context contains the index of the first matched character and the text that is surrounding the matched part.

Menu

There is an auto-hiding menu at the top of the window that proposes different ways to open RegExpresso's window.

  • :Opens RegExpresso in Internet Explorer's search pane. If you bookmark this link, it will open in the search pane every time you call it.
  • : Opens a new RegExpresso window sized like a toolbar, with the current options and result.
  • : Opens RegExpresso in the "_main" target, with the current options and result. With Netscape/Mozilla/Firefox and Internet Explorer, the "_main" target is to the current browser's window. With other browsers, this should be a new blank window, except if there is already a frame/window named "_main". This is intended to be used only from the sidebar. Since this is a standard link, and since it is updated every time the form is validated, one can bookmark it, or open it using the contextual menus provided by the browser, which may bring more control.
  • : Exports the result in a new empty window. In table mode, the output is CSV formatted. In text mode, the output is plain text. To save it, display first the source code of the page using your browser's menu.

URL parameters

As a customization effort, RegExpresso can handle arguments passed on the URL. Not only the options in the option panel can be passed on the URL, but also the following ones. To learn more about the encoding of the usual options in the URL, check the options you want in RegExpresso then copy the link that leads to the main window in the top menu.

In the following table, the "short name" is the name to use in the URL for this parameter.
For instance : http://regexpresso.berlios.de/regexpresso.html?highlight=on&stylesheet=http://toto.com/mystyle.css would lead to an instance of the tool with "highlight" checked and a custom stylesheet located at http://toto.com/mystyle.css.

Short nameDescription / values
stylesheet An URL to a stylesheet of type "text/css" to add to the tool. Since this is CSS, this stylesheet can override the style attributes already defined in the default one embedded in RegExpresso.
Any valid URL to a CSS stylesheet will do it.
empty In highlight mode, empty matches are replaced by the "empty set" character : ∅ (may not be correctly rendered here). This parameter can be set to change this default character.
Any character or string (preferably encoded in UTF-8) can be used.

Tips

RegExpresso saves the context in its URL

So, once you've executed a regular expression on some text, you can bookmark the link of the main window button to remember or to show the result to someone else. RegExpresso will execute the context when loading an URL with enough parameters. The only caveat is that the URL length is limited, so you won't be able to bookmark URL when the text to parse is too large (because it's passed on the URL as well).

Local installation

You can download the files for local use, but there are limits, mainly due to security restrictions.
Im' not going to list them all, but here are typical limits you'll find :

  • Opera : buttons "open in main window" and "open a new window" won't work because the URL contains parameters ("?param1=...&param2=...") that Opera cannot understand on local files ("file://" protocol).
  • Firefox : button "open a new window" won't work from the sidebar because it is loaded from the context of the main window, which will not permit to open local files due to security reasons.
  • With some other browser that may use relative URL in the case of local files, there may be security locks that prohibit opening such an URL from the sidebar, depending on the context of the current main window.
